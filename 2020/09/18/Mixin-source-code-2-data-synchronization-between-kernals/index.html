Mixin Kernals 如何相互同步数据

在去中心化网络中，一笔交易完成的实际含义大概是，这笔交易的数据被足够多的节点（比如 51% 以上）验证并确认。验证一笔交易的前提是收到它，为此 Mixin Kernals 会不断的从别的节点那里接受交易信息，也把自己验证过的交易信息发给其他的节点。这样达成的效果是，节点间的数据基本同步。

但这里有一个小问题，如果一个节点不接受交易或不发送交易怎么办？理论上是可以的，比如把机器关了、把网线拔了，而且完全不妨碍他收到节点奖励。
