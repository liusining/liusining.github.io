Mixin 源代码分析①：软件服务架构

Posted on 2020-09-15 | Edited on 2020-09-16
听别人信心满满的宣讲，也许不如自己安安静静地看代码

创新需要理解底层原理。大多数想创新的人应该会同意这句话。即使无法完全理解，多深入一分，视野也会成倍开拓。

本系列文章是我看 Mixin 代码的一些积累，希望帮助和我一样的第三方开发者理解其底层工作方式，让自己的视野超越官方 API 文档所定义的调用范式。

不过，Mixin 的代码在不断更新，所以请注意文章发布的时间。如果有与现状不符的地方，还请自行研究，但总体上应该还是能帮你了解 Mixin。

服务架构概述

我们先对 Mixin 的整体软件架构做一个简要（所以可能不精确）的描述。

Mixin 的核心部分是 Mixin Network，它目前包括 Mixin Domains 和 Mixin Kernals。Domains 控制或通过多签地址控制比特币等各种资产，作为数据出入 Mixin Network 的“关口”1。Kernals 就是 Mixin Network 上的全节点，他们基于 Domain 所提供的原始数据，记录用户在 Mixin Network 内部产生的“交易”1。

但对于 Mixin Messenger 的用户和其上的 Bot 开发者来说，这些软件服务单元是不可见的，因为他们被 Mixin 官方服务器封装起来了。目前大多数情况下，我们都是调用 Mixin 官方的 API 来间接使用 Mixin Network。

同时，Mixin Messenger 还有社交功能，他们也由 Mixin 官方服务器提供服务，所以 Mixin 肯定还有很多常规的、非区块链的网络服务单元，比如各种数据库、CDN 之类的。

Mixin 官方服务器上的代码没有开源，但为了完成对 Mixin Kernals 的调用，我们可以大致猜测 Mixin 官方服务器上做了哪些事。因此我们接下来几篇研究 Mixin Kernals 的调用过程和原理。
